Unity 编辑器拓展

基础

Editor 文件夹

在Unity中存在特殊的几种文件夹：

  1. Editor
  2. Resources
  3. StreamingAssets
  4. …

这里主要介绍Editor，这个文件夹的主要功能是 提供一个给Editor代码运行的环境，当游戏打包时不会将该环境下的代码编译打包进游戏中。编辑器启动时，在该文件夹中的代码就能开始启动了，不需要点击Play。

Editor文件夹不是指某一个文件夹，而是指所有在项目工程内所有命名为Editor文件夹。即同一个工程内，可以有多个Editor文件夹，路径任意，只需要名称为Editor即可。

细心的朋友能注意到，Editor与游戏逻辑代码是不同的两个Assembly。

MenuItem

MenuItem 属性用于向主菜单和监视面板上下文菜单中添加菜单项，它能够将任何静态函数转变为菜单命令。

Example

[MenuItem("EditorExtension/Menu/Hello Unity")]，这个 Attribute 需要引用 UnityEditor 的命名空间。用于拓展Unity的菜单栏。

EditorExtension/Menu/Hello Unity是菜单的路径，也可以添加到默认的菜单中如 Component/UnitEditor

public static void HelloUnity()为点击该选项后的调用函数，需注意的是函数必须是静态的。

Function: ExecuteMenuItem

有的时候我们需要调用其他 MenuItem 里的功能，但又不想直接调用相关的 API (例如当某个 API 在程序集中是不对外开放的)。

这时就可以使用 ExecuteMenuItem 方法，它通过 MenuItem 的路径进行调用。

Example

Set ShortCut

如果需要为 MenuItem 的功能添加快捷键，只需要在原先的路径后空一行，然后填入快捷键的符号即可。

如果快捷键中没有包含特殊字符，例如 #, &, %，则需要在快捷键的前面填写一个下划线 "_"。

不过需要注意的是，自定义的快捷键必须是之前没有使用的，不然就不会起效，Unity 会提示说按键重复。

MenuItem 快捷键对照表

Symbol Key
# Shift
& Alt
% Ctrl / Command
_a-zA-Z a-zA-z
